    GDBLinuxCPP

    Start

    gdb or gdb -q
# exit
ctrl + d or quit

    Use

    g++ -g main.cpp -o main
gdb main -q 
or
gdb -q 
file main

    Debug runing program

    ps ux | grep showpng 
gdb showpng 28334 -q

    CoreDump

    ** Check if off **

    ulimit -c
0 # is off
# tmp use 
ulimit -c unlimited

ulimit -c
unlimited # turn open

    permanent use

    vim /etc/security/limits.conf
# add line 
*       soft    core    unlimited
# and 
echo "core.%e.%p.%t" >/proc/sys/kernel/core_pattern

    gdb core

    gdb core* -q
# or
gdb -q 
core core*

    List

    1. list num

    gdb main -q
list 7
# or
l 7

    2. list func

    l main

    3. list range

    list 1,20

    Debug

    1. start start set temp-block

    start

    2. next run to the next line

    next
# or 
n

    3. next num

    # run 3 rows continuously
n 3

    Step

    Run the next line of code

    step
# or 
s

    Break

    1. Set break

    break 10
# or
b 10

    2. See break

    # info b
i b

    3. you can use ‘run’ or ‘r’

    run
# or
r

    4. delete break

    #d num
d 1

    5. tempbreak

    tb 100

    6. continue

    #continue run program
c

    backtrace

    1. Print function stack frame, trace back the entire call process

    bt

    2. Partial printing

    bt full

    3. current function stack frame

    i frame

    4. choose function stack frame

    bt
f 1

    5. Print function local variables

    info locals

    Value

    1. Print value and modify value(p & set) int i = 0

    p i
# 0

set var i = 10
p i
# 10

    2. View value type

    ptype error

    Thread status

    1. Info threads

    i threads

    2. Print specified thread stack

    thread apply 3 bt

    3. Print all thread stack

    thread apply all bt

    Use-GUI

    gdb file -tui -q
